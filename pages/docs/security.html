Security Practices

How we protect your data and infrastructure

Data Protection

  • Encryption at Rest: All stored data is encrypted using AES-256.
  • Encryption in Transit: All API communications use TLS 1.3.
  • Local AI Processing: Ollama runs locallyâ€”your data never leaves your infrastructure.
  • Secrets Management: API keys and credentials stored in encrypted environment variables.

Access Control

  • JWT Authentication: Secure token-based authentication with configurable expiration.
  • Role-Based Access: Admin, Operator, and Viewer roles with granular permissions.
  • Session Management: Secure session handling with ability to revoke all active sessions.
  • Rate Limiting: Protection against brute force and API abuse.

Infrastructure Security

  • Container Isolation: Each component runs in isolated Docker containers.
  • Non-Root Execution: All services run as non-privileged users.
  • Network Segmentation: Internal services communicate over private networks only.
  • Dependency Scanning: Regular vulnerability scans of all dependencies.

Audit & Compliance

  • Comprehensive Logging: All API requests and actions are logged with timestamps.
  • Audit Trail: Complete history of who approved/rejected which actions.
  • Distributed Tracing: Request correlation across services for debugging.
